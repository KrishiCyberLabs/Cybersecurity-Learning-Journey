# Offensive Security Intro

## Overview

In this room, I was introduced to the core concepts of offensive security and ethical hacking. The exercises demonstrated how security professionals identify weaknesses in systems and applications before they can be exploited by malicious actors. It provided a practical look at the mindset and methodology used during security assessments.

## Topics Explored

* Offensive Security Fundamentals
* Ethical Hacking Principles
* Reconnaissance Techniques
* Website Enumeration
* Hidden Resource Discovery
* Security Testing Methodology

## What I Learned

One of the biggest takeaways from this room was the importance of reconnaissance. Before attempting to identify vulnerabilities, security professionals spend significant time gathering information about their target. I learned how hidden pages, directories, and publicly available information can reveal valuable insights during an assessment.

The room also reinforced the idea that offensive security is not about attacking systems without permission but about proactively finding and fixing weaknesses in authorized environments.

## Skills Developed

* Reconnaissance and Information Gathering
* Website Enumeration
* Basic Security Assessment Techniques
* Analytical Thinking from an Attacker's Perspective
* Understanding Security Testing Workflows

## Tools and Concepts

* Web Browsers for Investigation
* URL Analysis and Manipulation
* Directory and Resource Discovery
* Security Assessment Methodologies

## Personal Reflection

This room helped me understand how ethical hackers approach security assessments and why reconnaissance is often considered the foundation of successful testing. It showed that even small pieces of information can contribute to a larger understanding of a target environment. Overall, it provided a strong introduction to the offensive security mindset and strengthened my interest in learning practical cybersecurity skills.
